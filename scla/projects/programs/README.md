# `projects/programs/` — active program development

Work in progress on new or evolving programs. **This is not the same as `scla/programs/`**, which is the pipeline-generated knowledge base reference for existing programs.

| This directory | `scla/programs/` |
|---|---|
| Human-managed development work | Pipeline-generated KB entries |
| New programs being built | Existing programs, documented |
| Editable freely | Overwritten on pipeline re-run |

## What belongs here

- New program proposals and planning docs
- Program redesign or iteration work
- Pilot documentation

## Key sources to reference

- `scla/source-of-truth/program-names.md` — canonical names (lock in here before using anywhere)
- `scla/source-of-truth/decisions-log.md` — prior program decisions
- `scla/programs/` — reference existing programs to avoid duplication

## Status lifecycle

`draft` → `active` (in development) → `complete` (launched) → `archived`
