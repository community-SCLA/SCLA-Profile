Core Model

Core Model

Micro‑projects Model:  10–40 hour scoped projects from real organizations, completed remotely, with a concrete deliverable (e.g., market research memo, basic data analysis, slide deck, content calendar). These are true micro‑internships that sit between a project and a short freelance engagement.

Simulated Sprints: 4–8 hour self‑paced “job simulations” using realistic but safe tasks from a named company or industry, with structured instructions and rubrics. These work well as on‑ramps for students who have little experience or want to test different career paths.

Cohort Accelerator: 3–6 week part‑time virtual experience with a mix of mentor sessions, skill workshops, and a capstone project for an employer partner. This feels more like a compressed internship, with peer community and coaching layered on top.

Position:  Short, scoped projects (micro‑internships) nested inside business-backed cohorts to provide real employer projects into credit‑aligned, coached micro‑internships.

What’s “Real” About the Work

High-fidelity Business Projects. Real companies, real deliverables for startups, SMBs, large organizations, nonprofits, or governmental agencies, where outcomes matter, timelines are real, and the work has a visible impact.

Hybrid Learning. Real client context and goals, but narrowed scope, pre‑determined tasks, pre-cleaned data, and more scaffolding (templates, examples, structured briefs) so interns or intern teams can deliver reliably in 2–4 weeks.

Live Wrap-around Support. Strong support services and events to help interns prepare, skill-up, or connect and receive deeper dives into the project and companies.

Value Prop

For students:

1–3 portfolio artifacts, tied to specific skills and tools.

Practical, applicable outcomes to drop into a resume or LinkedIn profile.  a supervisor or mentor blurb/rec, and language for college essays.

Partner-level feedback and LOR to quote in applications or request as formal references.

For employers:

Low‑lift brand exposure + vetted, scoped projects that don’t break anything, with a micro‑talent pipeline

Low‑risk access to student talent via tightly scoped projects that actually move something forward (research, early‑stage analysis, outbound lists, content drafts, etc.).

A playbook/checklist that makes participation low‑lift (clear brief, async touchpoints, standard feedback form).

Structural Guardrails

Duration & Load:

Common sweet spot: 2–4 weeks, 5–10 hours/week for a micro‑project; over 2-6 weeks accounting for busy schedules or breaks;

Matching & Access:

Open project-matching (students get matched to 3 projects - choose one using AI-generated results from assessment)

Individual vs. team projects; teams can mirror real workplace collaboration but individual projects can be completed easier and more efficiently

Asynchronous, individual vs1 - look at metrics to determine uptake of potential team projects

Support & Scaffolding:

Clear project briefs, milestone check‑ins, example past deliverables, and a simple communication protocol with SCLA team

Light coaching/office hours from SCLA team to keep quality and completion rates high.

Key workflows (end‑to‑end)

Company Project Creation & Approval

SCLA team creates a Project via form with required fields and templates (scope, deliverables, hours, skills, timeline)

Validation rules: required fields, hour range (e.g., 10–40), no disallowed content, etc.

Internal review/approval queue: admin can edit, approve, or request changes.

On approval, project is:

Saved in project database.

Synced to CRM as an “opportunity/engagement” with links to company and contact.

Published to student based on their “match”

Student Onboarding & Matching

Student profile creation (already done but may need to ask for more information): collect key fields, resume/CV upload, skills, interests, availability.

Pre‑program video modules gated as prerequisites (??) for applying (e.g., professionalism, communication, basic tools)

Project discovery: students can search/filter projects by skills, project scope,  company type.

Application flow: student submits short assessment / application (motivation, skills, interest.

Matching logic:

v1: manual review by staff with simple filters;

optional v2: automated rules (e.g., only students with required modules completed, skill tags match, capacity not exceeded)

Acceptance: when accepted, system creates Enrollment, triggers notifications/ emails to student + team, and creates project workspace in LMS.

Project Execution & Communication

On kickoff:

Auto‑generate checklist/milestones in LMS project space (e.g., kickoff call, draft, final deliverable)

Event system: schedule kickoff event (calendar invites + virtual link) and tie attendance back to Enrollment.

During project:

Communication: define whether you use in‑platform messaging, email, or integrated channel chats with logging

File sharing: submit drafts and final deliverables through LMS assignment objects.

Milestones / Progress tracking: status updates (not started/in progress/at risk/completed), auto reminders if milestones overdue

Completion, Feedback, and Credentials

Student submits final deliverable via LMS; staff reviews and marks as accepted/needs revision.

Staff / Employer fills short evaluation form (Likert + comments) tied to Enrollment

Student completes reflection survey; data stored for outcomes reporting

System:

Marks project as completed.

Issues certificate/badge/micro‑credential if criteria met (optionally linked to micro‑pathways or stackable credentials)

Syncs key fields back to CRM (e.g., project completed, evaluation scores, employer feedback)

Security, Roles, and Permissions

Roles: Admin, Staff/Coach, Student, (optionally Advisor).

Permissions matrix:

Students see their own profile, eligible projects, their project spaces, and anonymized employer feedback.

Staff / Employers see only their projects and assigned students, not other companies’ data.

Staff see global view, can override matches, edit projects, impersonate for support.

Data protection: ensure access control between LMS, CRM, and event tools, and log key actions

Data & Reporting Requirements

Program health: number of active companies, active projects, applications per project, match rates

Student outcomes: completion rates, average evaluation scores, skills gained (from rubrics), follow‑on internships/jobs (where known).

Employer outcomes: satisfaction scores, repeat project rate, time‑to‑fill

Operational: time from project submission to approval, time from approval to match, issues/tickets
